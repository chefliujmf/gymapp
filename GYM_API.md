# Gym workout interchange (cyclingcoach → gymapp)

The plan lives on **intervals.icu**. gymapp executes it. To put a gym workout on a
day, write a **planned event** to intervals.icu with a structured description that
gymapp can parse. Both gymapp and the cyclingcoach use the **same format**, so the
coach can push gym sessions exactly like it pushes rides.

## Transport — intervals.icu REST

```
POST https://intervals.icu/api/v1/athlete/{athleteId}/events
Authorization: Basic base64("API_KEY:" + <api_key>)
Content-Type: application/json

{
  "start_date_local": "2026-06-20T00:00:00",
  "category": "WORKOUT",
  "type": "WeightTraining",
  "name": "Leg Day",
  "description": "<gym block — see below>"
}
```

`DELETE …/events/{id}` removes one. `GET …/events?oldest=YYYY-MM-DD&newest=YYYY-MM-DD`
lists them (gymapp reads today's from here).

## Description format

```
[gymapp] 3 rounds
• Squat [e-30993] — 40s / 15s rest
• Reverse lunge [e-31040] — 40s / 15s rest
• Plank [e-31112] — 30s / 20s rest
```

Each exercise is either **timed** (countdown, auto-advance) or **sets×reps**
(log weight×reps per set, manual advance, rest between sets):

```
[gymapp]
• Back squat [e-30993] — 4×5 @ 80kg / 120s rest      ← sets × reps @ weight / rest
• Pull-up [e-31040] — 3×8 / 90s rest                 ← weight optional
• Plank [e-31112] — 40s / 15s rest                   ← timed
```

Rules:
- **Header**: `[gymapp]` marks it as a gymapp-readable gym workout. Optional
  ` N rounds` sets repeats (default 1). gymapp's own builder may emit
  `[gymapp:<templateId>]` for a local fast-path — the coach should just use `[gymapp]`.
- **Each exercise line**: `• <name> [<exId>] — <body>`
  - **timed body**: `<work>s` optionally ` / <rest>s rest`
  - **reps body**: `<sets>×<reps>` optionally ` @ <weight>kg` and ` / <rest>s rest`
  - `<exId>` is optional but recommended — it links to the exercise library so the
    demo video/image show in the player. Pick ids from `gym-exercise-index.json`.
  - Bullet may be `•`, `-`, or `*`; the dash may be `—`, `–`, or `-`; `×` or `x`.

## Exercise library

`gym-exercise-index.json` (committed, regenerated by `npm run build:catalog`) is the
pickable list:

```json
[ { "id": "e-30993", "name": "Squat", "category": "Legs" }, … ]   // 1993 entries
```

The coach selects exercises by `id` and emits the lines above. gymapp resolves each
`id` to its video/image when playing.

## Round-trip

`src/intervals.ts` has `encodeGymWorkout(spec)` / `parseGymWorkout(description)` —
the canonical reference implementation. Any producer that emits the format above is
compatible.
