# Platform-first — keep the user's attention IN Platyplus

**JM directive (2026-07-22):** "all decisions must be in accordance to drive and keep user attention in the platform."
Platyplus is the product; intervals.icu / Strava / the Garmin are **mirrors and rails**, not the home. Every choice
should make Platyplus the place the athlete opens, reads, and acts — and treat every external surface as a **doorway
back in**, never a destination that replaces it.

## The rule
When a decision affects what the user sees or where they act, ask: *does this pull attention INTO Platyplus, or leak it
out?* Choose the option that keeps them in the app (or brings them back). If an external surface must carry content
(intervals event, Strava post, a push notification), the **Platyplus entry point comes FIRST and is unmistakable.**

## Concrete applications (from JM's calls)
- **Order matters — the Platyplus link/CTA goes at the TOP.** On an intervals/Strava/pushed item, the "open it in
  Platyplus" link + ownership note lead the description, not buried at the bottom (#736). The first thing they read is
  the way back in.
- **Give the outside surface *less*, the app *more*.** External copy is a scannable teaser + a link; the full plan,
  meals, mind, insight, history and actions live in Platyplus. Never reproduce the whole experience outside (that trains
  the user to live there instead).
- **Notifications point inward.** Every notification deep-links to the relevant Platyplus screen; the bell is a reason to
  open the app, not a self-contained feed (#733). "What's new" surfaces new capability so they come see it (#732).
- **Own the plan, don't let intervals become the editor.** Platyplus is the master; edits made outside are reverted, and
  the copy says so — but framed as "edit it in Platyplus," i.e. come back, not "don't touch this" (#588/#726).
- **Coach text sends them home.** A pushed workout's note ends/loops toward the app for the full brief, cues, fuelling and
  the coach chat — the depth is inside.
- **Reduce reasons to leave.** If a task can be done in-app (log weights, edit a session, mark a rest day, see stats),
  build it in-app rather than sending the user to another tool.

## When it applies
Any decision about: link placement + wording, description/notification/email copy order, what content goes on an external
surface vs stays in-app, new features (build in-app vs offload), nav that would send the user elsewhere, and coach-authored
public/pushed text. If a change would make an outside surface more compelling than Platyplus, that's the smell — invert it.

## The test
Before shipping anything the user reads outside the app: *is the Platyplus doorway the FIRST and clearest thing, and does
the outside copy make them want to open Platyplus rather than settle where they are?* If not, reorder/rewrite until it is.

Ties: `options-first` (mock the ordering), `platyplus-coach-public-text-voice` (the wording), `whole-app-audit`
(check every external touchpoint leads inward), memory `platyplus-integrations` (intervals = hub, not home).
