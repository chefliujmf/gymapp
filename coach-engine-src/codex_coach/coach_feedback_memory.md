# Coach Feedback Memory

> **Owns:** durable, distilled coaching rules/hypotheses and whether past advice worked. Read before analysis, write after feedback. Raw per-workout entries live in `feedback_log.md`; note structure in `coach_feedback_format.md`.

This file is the durable coaching feedback database.

It tracks:
- athlete feedback about the coach's advice, wording, and usefulness;
- coaching hypotheses that should be tested against future workouts;
- whether a recommendation helped, did nothing, or created friction;
- the next adjustment the coach should make.

Use this file before:
- analyzing a completed activity;
- writing an Intervals.icu coach note;
- regenerating a plan;
- changing nutrition, supplement, mental-focus, Centr, strength, or recovery guidance.

Do not paste this file into activity notes. It is coach-memory, not athlete-facing feedback.

## Update Rules

- Add a row when the athlete says a recommendation was useful, confusing, too vague, too much, too little, or not aligned with goals.
- Add a row when a workout result confirms or contradicts a coaching assumption.
- Mark status as `active`, `testing`, `validated`, `changed`, or `retired`.
- When a rule changes, keep the old row and add the adjustment rather than deleting the trace.
- If a recommendation is repeated for 2-3 workouts and the athlete feedback or data does not support it, revise it.
- If the athlete marks text with `***`, treat it as coach-improvement feedback and log the rule change here or in the linked instruction file.
