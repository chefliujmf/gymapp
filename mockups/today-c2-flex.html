🧩 The model (why it stays flexible)
Each row is a typed block from the day's data. Order + which blocks show is data-driven.

type → body renderer (sets×reps · power · pace · laps · class). N per day.

➕ New type later (hydration, mindset, labs…) → add a renderer once, it appears here.

Yes — fully flexible. Swimming = a new workout body; supplements = a sub-block of Fuel; sauna = a Recovery block. The check-in/readiness stays the universal top. Adding a new section or sport later is a data + one-renderer change, not a redesign.
