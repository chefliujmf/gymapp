Platyplus honors its core principle — safety & dose enforced in code, not prompts — to a world-class standard on exactly one axis: endurance intensity for an adult ride/run athlete. Everywhere else — the whole gym axis, all of swimming, triathlon, and every vulnerable persona (pregnant · postpartum · teen · masters) — those same rules are prompt-only, and several defaults are inverted toward danger. It's a 4/5 product for the male cyclist it was born from, a 3/5 for most personas a launch will onboard. That gap — not any single bug — is the score.

Top findings · ranked by severity × impact
1Critical

Swim shape-clamp is a complete no-op for real swim plans

swimmer

create_swim emits only {time, distance, load} — zone discarded — and swim plans store segments:[], so enforceShape clamps nothing. Every swim intensity/safety limit (incl. pregnant/teen swimmer) is prompt-only. A green unit test hides it by hand-injecting power segments create_swim never produces.

Fix → emit per-set zone→%CSS as a segment-equivalent and clamp at save under intensityCeiling; rewrite the test to feed real create_swim output.
2High

Beginner default is the MOST aggressive week (2 quality, VO2 ceiling)

beginner

weekShape only softens on a "maintain" keyword; a blank or ambitious goal falls to build/2/VO2 with nothing clamping it. The "conservative ramp-in" is prompt-only.

Fix → make the conservative shape the DEFAULT when there's no load history: flat, ≤1 quality, ceiling ≤ tempo, ramping only as absorbed load accrues.
3High

Postpartum with no birth-date degrades UNSAFELY to a full build

postpartum

The graded-return path only fires if postpartumSince is set (an optional field). Turn "Pregnant" off without a date → coached as normal → 2 quality + VO2.

Fix → on pregnant true→false, auto-set a provisional postpartumSince=today so clearing pregnancy enters the graded return by default (opt-out, not opt-in).
4High

Pregnant gym: no supine/T2+ contraindication is code-enforced

pregnant

The gym assembler gets no pregnant/trimester arg → resolves flat-supine bench/floor-press/dead-bug for a T2+ athlete, then the prompt says "build them exactly", overriding the "no supine" rule 30 lines above it.

Fix → pass pregnant+trimester into the assembler; at T2+ filter supine/prone patterns + add an enforcePregnancyGym save-clamp peer to enforceTeenGym.
5High

Postpartum clamps INTENSITY but not IMPACT — the real risk is un-gated

postpartum

Early-postpartum is clamped to 75% pace, but pelvic-floor risk is ground-reaction impact, not aerobic intensity — a ceiling-compliant easy run is still contraindicated pre-rebuild, and modality is never downgraded.

Fix → return an impact directive from weekShape (none <6wk, walk-run 6–12wk) and enforce it by converting/blocking run modality, not just clamping %.
6High

Triathlete collapses to bike-only in three code layers

triathlete

['triathlon'] matches none of cycling/running/swimming, so archetype variety, dev-priorities, and the save clamp all fall through to "ride" + a flat global-2 cap. 2 of 3 sports get zero code service.

Fix → make archetype/systemPrompt/weekShape triathlon-aware: per-discipline archetypes, all three focus lists, ~1 quality/discipline by phase, clamp the lowest-priority discipline.
7High

Teen gym derates rep count but not LOAD — still ~85% 1RM

teen

enforceTeenGym only floors reps<5→5; clampMainReps forces support mains to 3–6 @ 80–90% 1RM — the two passes combine to lock a minor into the heaviest zone, opposite of NSCA youth guidance.

Fix → a teen branch that caps %1RM (≤70–75%) and forces a technique band (≥8 reps), overriding the support down-clamp.
8High

Concurrent-training separation is prompt-only — heavy legs can precede a key ride

all endurance+gym

maxPerDay stops same-day stacking, but nothing prevents an adjacent-day interference collision — the defining requirement for every strength-supported endurance athlete.

Fix → a unit-tested check in reenforceShapeAll that detects heavy gym ±1 day from a quality endurance day and flags/moves it.
9High

Bodybuilder under-doses volume ~½ the engine's own 10–20 target

bodybuilder

set_gym_target advertises 10–20 sets/muscle and Stats judges against it, but the assembler plans ~6 — half the minimum effective hypertrophy volume, never scaled by focus. A say/do gap.

Fix → scale sets-per-primary + a 2nd movement on priority muscles for focus=muscle; unit-test weekly sets/muscle ≥ 10.
10High

Female menses cut a quality day — contradicts the engine's own science

female-cycle

weekShape de-loads the menstrual phase with no symptom check, but cycle.js rates it near-neutral and the female engine calls it a green light for PRs. The graded phase load modifier is never multiplied into any target.

Fix → restrict the cut to late-luteal (menstrual only if symptomatic); fold the phase modifier into weeklyLoadBudget so it's code-enforced.
11High

Gym never progresses week-to-week — no code periodization or deload

all gym

periodization.js is endurance-TSS only; the assembler hardcodes sets and emits the same reps every week. Double-progression + deload exist only as prose the fixed frame overrides.

Fix → a gym meso-cycle (accumulation→intensification→deload) feeding week index into the assembler so reps/%1RM ramp and deload cuts volume ~40–50%.
12High

App-wide: dead routes, split feedback stores, off-theme hex

ux · nav · stats · admin

Two disconnected feedback stores drop/re-prompt feedback; /more (Eat & Mind) is a dead route; the Cycling stats page is near-empty & gates out manual-FTP cyclists; a blind navigate(-1) dead-ends on deep-links; invented hex (incl. the exact #3d7bff blue the theme memory exists to kill) recurs in admin/stats.

Fix → mirror plan feedback into activityFeedback + unify the client store; fix/hide /more; give Cycling stats parity + a manual-FTP path; guard the back-button; sweep hex → theme tokens.
Fix this first

Close the code-enforcement gap on safety — before any new capability.

The product's whole thesis is "prompt-only rules get ignored, so decide in code." That's honored for adult endurance intensity and abandoned everywhere a vulnerable or non-cyclist athlete lives. Workstream one is a code-enforced safety floor: (a) flip the inverted defaults so absence-of-input yields the conservative shape — beginner-no-history and postpartum-no-date must never resolve to 2× VO2 / full build; and (b) make the flagship save-time clamp actually run for swimming and for the gym (pregnancy · postpartum · teen-load). For a product onboarding whoever signs up, a dangerous default is worse than any missing feature.
