Split the mechanical build (deterministic, code) from the coaching voice (light LLM pass). Only for gym-MAIN athletes; endurance untouched.

1
Assemble the week CODEThe existing #718 engine builds this athlete's individualized gym week — split · sets/reps by GOAL (strength 3-5 heavy · hypertrophy 6-12 · general) · meso-progression · owned equipment. Already the source of truth.
2
Resolve exercise IDs CODEMatch each move name → a real catalog exId (same matcher the /api/exercises endpoint uses, server-side). No LLM, no 30-call search loop.
3
Save the sessions CODEWrite all N gym days via upsertPlan (warm-up/cool-down/main, sets/reps/tempo/each-side). The plan is now GUARANTEED built — the coach can't loop or fail it.
4
Coach adds the voice LIGHT LLMA focused pass: "these sessions exist — add a one-line form cue per lift + objective/success/fuel per session. Do NOT change the exercises/sets/reps." Can't fail the structure; just the human coaching layer.

Why this isn't the removed skeleton (#516): #516 was a GENERIC endurance rotation forced on everyone. This is (a) gym-only — endurance stays fully LLM-authored; (b) genuinely per-athlete (the #718 assembler reads their sport/goal/equipment/history); (c) the coach still individualizes the voice. Code does the reliable mechanical part; the LLM does what it's good at.
